# Guia de Contribuição

As contribuições para o projeto BackBet são restritas aos colaboradores autorizados.

## Regras básicas

1. O acesso ao repositório é concedido apenas a membros da equipe técnica.
2. Toda modificação deve seguir o fluxo de revisão:
   - criar branch descritiva
   - abrir pull request
   - aguardar revisão de pelo menos um revisor técnico
3. Commits devem seguir convenções claras:
   - `feat:` novas funcionalidades  
   - `fix:` correções  
   - `docs:` documentação  
   - `refactor:` melhorias internas  
   - `chore:` ajustes não relacionados a código  
4. É proibido incluir dados sensíveis:
   - tokens  
   - credenciais  
   - arquivos .env  
   - informações de produção  

## Qualidade e padrões

- Manter código seguro, limpo e auditável.
- Priorizar consistência entre serviços.
- Testes são obrigatórios para novas funcionalidades backend.
- Testes devem ser feitos de forma compusiva.

Dúvidas ou necessidade de acesso devem ser enviadas para a liderança técnica.
